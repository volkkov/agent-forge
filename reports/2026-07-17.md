## Flagged as hype

- **Legal-Swarm-Template** — The README describes a concrete mechanism (five named agents on the Swarms framework, run via main.py with an API key) so it is not purely imaginary, but it is dominated by unverifiable marketing: 'revolutionary', a cost-savings table claiming 99.97% reductions with only '*Costs based on average legal fees and API usage' as attribution, and security claims with no supporting detail. Metadata shows only 17 stars, 5 forks, created Nov 2024 and last pushed Dec 2024 — recent enough to avoid a 'broken' label but thin maintenance. The 'One-Click' description is inaccurate (requires clone, pip install, env setup). Overall, more hype than evidence of a production-ready tool.
- **radiology-swarm** — The README is largely marketing copy: it uses terms like 'powerful, enterprise-grade', '>99.9% accuracy', '<2 seconds' with no reproducible benchmark, dataset, or methodology. The 'Status' section lists CI/CD, automated testing, and documentation as unchecked TODOs, directly contradicting the Tests badge and 'stable version 1.0.0' claim. Despite recent push (2025-10-13) and MIT license, the concrete mechanism is just a thin wrapper around OpenAI calls with no real medical validation, making the enterprise/HIPAA claims unsubstantiated fluff.
- **klaatcode** — The README describes a concrete mechanism (per-request routing to 5 tiers via hosted Klaatu service, code-graph index, reproducible bench via `bun run bench`) and metadata shows recent push (2026-07-17), 83 stars, active CI badge, not archived — so it is not broken. However, the core differentiation ('Claude Code-grade accuracy', '10x cost cut', '5.5x cheaper') depends entirely on a closed hosted backend (Klaatu at klaatai.com) that requires account login; the 'open source' CLI is explicitly a thin client to a proprietary service, making the headline cost/accuracy claims unverifiable without the gated backend. This is a real tool but the marketing framing ('10x', grade-level claims) overshadows the actual open-source contribution, which is a terminal client to a SaaS.
- **arifos** — The README describes a concrete mechanism (MCP kernel with 13 floors, 8 ABI capabilities, CI badges, PyPI package) and shows recent activity (last pushed 2026-07-17, CI workflows, live version manifest), so it is not 'broken'. However, the description and README are saturated with unverifiable, grandiose framing ('Constitutional MCP kernel', 'the LAW', 'sovereign agentic era', 'AGI/ASI/APEX trinity', 'forged not given') and the project is explicitly a public snapshot behind a private live deployment owned by one person with 'absolute veto'. The concrete utility is a governance gate for MCP tool calls, but the marketing/mythology layer dominates and the open repo is not the real system. This fits 'hype' over 'useful'.
- **automated-crypto-fund** — The repository contains no technical documentation, code explanations, or architectural details in the README, only social media links. The claim of 'eal-money' trading without any verifiable mechanism is a high-risk marketing buzzword.
